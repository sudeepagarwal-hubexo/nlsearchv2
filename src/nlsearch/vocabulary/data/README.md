# Vocabulary / ontology datasets

Generated by `scripts/sync_ontology.py` from **Mimir gold dimension tables** (Databricks).

## Files

| File | Source table | Used for |
|------|----------------|----------|
| `contract_stages.json` | `contract_stages` + translations | Tender / construction procurement NL |
| `planning_stages.json` | `planning_stages` + translations | Planning-stage NL |
| `project_statuses.json` | `project_statuses` + translations | Default exclusions (CAN, HOL, …) |
| `contract_types.json` | `contract_types` | Design-and-build, traditional, … |
| `development_types.json` | `development_types` | New-build, refurbishment, … |
| `building_use_groups.json` | `building_use_groups` | Sector → BUG-* mapping |
| `building_use_definitions.json` | `building_use_definitions` | Fine-grained building use (BU-*) |
| `role_definitions.json` | `role_definitions` + translations | Architect, client, electrical contractor |
| `role_groups.json` | `role_groups` | Role group prefixes (MCT, ARC, …) |
| `green_building_schemes.json` | `green_building_schemes` | BREEAM, low-carbon semantic |
| `document_types.json` | `document_types` | Permits, drawings, RFPs |
| `ownership_types.json` | `ownership_types` | Public/private sector |
| `facet_*` | DISTINCT on `project_fields` / `site_address` / `project_roles` | Values actually in use |
| `facet_places.json` | `site_address` | Gazetteer (postal_town, regions) |
| `facet_companies.json` | `project_roles.company_name` | Company name resolution |

## Regenerate

```bash
python scripts/sync_ontology.py
```

Requires `NLSEARCH_DATABRICKS_*` and warehouse access.
